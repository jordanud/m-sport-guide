# m-sport-guide

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面，内容涵盖运动相关的信息、指南与参考资源。所有页面均为静态 HTML 格式，不依赖后端服务，可直接通过浏览器访问。

这是一个通用型归档仓库，不针对任何特定网站或域名，旨在提供一个结构清晰、易于维护的页面集合。

## 目录说明

```
m-sport-guide/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（如 CSS、图片、字体等）
├── docs/           # 文档或附加说明（可选）
└── README.md       # 本文件
```

- `pages/`：页面主体目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面所需的样式表、图像、脚本等公共资源。
- `docs/`：如有需要，可放置使用说明、更新日志等文档。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接打开或部署到任意静态托管服务。
- 页面之间无强依赖关系，可单独更新或替换。
- 文件名建议使用英文小写，以短横线分隔（如 `guide-basic.html`），便于识别和链接。
- 页面内容应保持简洁、准确，避免包含过时或不可靠的信息。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来修正错误、优化内容或新增页面。
- 新增页面时，请遵循现有目录结构，并在 `pages/` 下创建新文件。
- 若涉及资源文件，请放入 `assets/` 对应子目录中，并在页面内使用相对路径引用。
- 请勿在页面中加入任何形式的跟踪脚本、广告或外部链接指向特定商业网站。
- 本仓库以归档和分享实用信息为目的，不承担任何内容准确性的法律责任。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/m-sport-guide.git
   ```
2. 直接打开 `pages/` 下的任意 HTML 文件即可浏览。
3. 也可将整个仓库部署到 GitHub Pages 或其他静态托管服务，获得在线访问地址。
